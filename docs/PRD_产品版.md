# AI CAD Agent 产品需求文档（产品版）

## 1. 文档信息
- 产品名称：工业级 AI 建模助手（AI CAD Agent）
- 文档版本：v1.0（基于当前上线版本）
- 文档类型：产品版 PRD（面向业务沟通、评审与路线规划）
- 目标读者：产品经理、研发负责人、业务/售前、项目干系人

---

## 2. 产品愿景与定位

### 2.1 愿景
让用户通过自然语言即可快速获得可用的 CAD 模型，实现“对话即建模”。

### 2.2 产品定位
一个具备参数协商、自愈修复和在线预览能力的 AI CAD 助手，适合作为工业智能建模场景的演示与早期生产力工具。

### 2.3 当前定位阶段
Demo 到可交付 PoC 之间：核心链路已打通，具备公网部署能力，正在向稳健性与标准化推进。

---

## 3. 背景与问题

传统 CAD 建模存在以下问题：
- 学习成本高，依赖熟练工程师操作；
- 前期概念验证效率低，需求变更成本高；
- 非标准输入（自然语言）无法直接转为可执行建模流程；
- LLM 生成代码稳定性不足，容易出现 API 与几何执行错误。

本产品通过多 Agent 编排 + 自愈机制 + 对话交互，解决“从描述到模型”的效率瓶颈。

---

## 4. 目标用户与核心场景

### 4.1 目标用户
- 机械设计工程师（快速起模）
- 研发/创新团队（方案验证）
- 售前/演示团队（概念展示）
- 轻量 CAD 使用者（低门槛建模）

### 4.2 核心场景
- 输入“做一个连杆，中心距150mm...”后快速生成模型；
- 输入“带8孔法兰盘”时系统自动补问或填默认参数；
- 执行失败时自动重试并生成可读错误反馈；
- 云端链接可访问、可下载 STEP、可在线预览 3D。

---

## 5. 产品目标与成功标准

### 5.1 产品目标（阶段）
1. 支持自然语言输入与多轮对话；
2. 自动完成参数理解、计划、代码、执行链路；
3. 支持执行失败自愈重试（最多 3 次）；
4. 支持 STEP 下载与网页 3D 预览；
5. 完成云端部署与密钥安全管理。

### 5.2 成功标准（建议）
- 关键路径成功率（STEP 生成）>= 70%（Demo 阶段）
- 单轮平均完成时长 <= 60 秒（视模型复杂度）
- 自愈挽回率（首次失败后重试成功）持续提升
- 用户对“可解释性（状态/错误）”主观满意度提升

---

## 6. 功能范围（当前版本）

### 6.1 对话输入模块
- 聊天式输入（`st.chat_input`）
- 会话历史持久化（刷新不丢失）
- 消息类型区分：用户 / Parser追问 / 系统通知

### 6.2 参数解析模块（Parser）
- 识别零件类型（连杆、法兰等）
- 提取参数、缺失项、置信度
- 提供澄清问题和建议默认值
- LLM 失败自动回退规则解析

### 6.3 建模计划模块（Planner）
- 基于需求 + metadata + RAG 输出建模计划
- 提供步骤化建模指令，供 Coder 执行

### 6.4 代码生成模块（Coder）
- 生成 CadQuery 代码
- 静态校验（导入、result、禁用API）
- 参数强制注入（避免变量未定义）
- 错误回注后重试修复

### 6.5 执行导出模块（Executor）
- 执行代码并捕获异常
- 成功导出 `part.step` + `part.stl`

### 6.6 动态看板与结果反馈
- 展示 part_type / confidence / missing_params
- 链路状态（Parser/Planner/Coder/Executor）
- 错误详情、历史轨迹、一键重试
- STEP 下载与 3D 预览

---

## 7. 交互流程（用户视角）

1. 用户输入建模需求（自然语言）  
2. Parser 解析参数：
   - 若关键参数不足：追问/建议默认值
   - 若可继续：进入 Planner  
3. Planner 产出建模步骤  
4. Coder 生成脚本并执行静态校验  
5. Executor 运行脚本并导出文件  
6. 失败则进入自愈循环，最多 3 轮  
7. 成功后用户下载 STEP 或在线查看 3D

---

## 8. 关键产品策略

### 8.1 参数协商策略
- 对几何关键缺参优先追问；
- 对可推断参数支持默认值；
- 在体验与准确性之间平衡“追问频率”与“自动放行”。

### 8.2 自愈策略
- 错误文本映射到修复提示；
- 结合上一轮错误定向改写代码；
- 防止无限重试（上限控制）。

### 8.3 可解释性策略
- 透明展示 metadata、状态、历史、错误；
- 用户可以理解系统为何失败/为何追问。

---

## 9. 非功能需求

- 安全：密钥不入库，环境变量读取，云端 Secrets 管理
- 可靠：错误捕获完整，链路可回退
- 可部署：依赖与系统库可复现
- 可维护：模块边界清晰，配置集中化

---

## 10. 风险与限制（当前）

- LLM 质量波动导致几何稳定性波动；
- 复杂零件场景仍可能失败；
- 3D 预览受浏览器网络/CDN策略影响；
- 不同云端 Python 版本可能影响 CAD 依赖安装。

---

## 11. 下一阶段优化路线

### P1（近期）
- 默认值使用前增加“确认一步”
- 法兰/连杆模板化建模骨架（提升一致性）
- 错误修复策略持续扩充

### P2（中期）
- 建立标准测试样例与自动回归
- RAG 由规则检索升级为向量检索
- 预览前端资源本地化（弱网更稳）

### P3（远期）
- 多零件/装配关系建模
- 企业级权限与审计
- 行业标准规则库深度融合

---

## 12. 发布与运营建议

- 对外发布采用“Unlisted/Public 链接 + 演示脚本”模式；
- 预置 3~5 个高成功率 Prompt 作为首屏示例；
- 收集失败案例与日志，形成每周错误闭环优化。
